# 方法說明

## 1. 問題定義

本研究將股票未來 h 日報酬方向視為二元分類問題。

若未來 h 日報酬大於 0，標籤為 1；否則標籤為 0。

CNN 模型輸出未來報酬為正的機率，並使用該機率作為股票排序分數。

---

## 2. 價格影像生成

每個交易日使用過去 window 日 OHLC 資料生成價格影像。

影像設計如下：

1. 每日 OHLC 使用 3 pixels 寬度。
2. Open 畫在左側。
3. High-Low 畫在中間垂直線。
4. Close 畫在右側。
5. 背景為黑色。
6. 線條為白色。
7. 視窗內含 NaN 則跳過該樣本。

---

## 3. 模型訓練

模型使用 CNN 進行影像分類，訓練目標為最小化分類損失。

主要訓練設定：

| 項目 | 設定 |
|---|---|
| optimizer | Adam |
| learning rate | 1e-5 |
| batch size | 128 |
| early stopping | validation loss |
| device | CUDA GPU |
| mixed precision | enabled |

---

## 4. 樣本外預測

模型訓練完成後，固定模型參數，在測試期間產生每檔股票每個再平衡日的預測分數。

測試期間資料不參與模型訓練與模型選擇。

---

## 5. 投資組合建構

每個再平衡日，依照模型預測分數將股票分成 D1 到 D10。

D10 代表模型最看好的股票，D1 代表模型最不看好的股票。

本研究主要觀察 D10-D1 多空組合：

1. Long D10。
2. Short D1。
3. 組內等權重。
4. 持有 horizon 個交易日。
5. 扣除交易成本。

---

## 6. 績效評估

本研究不只使用 AUC 評估模型，也檢查投資組合績效。

主要指標包含：

1. AUC。
2. D10-D1 年化報酬。
3. Sharpe ratio。
4. 最大回撤。
5. 勝率。
6. t-stat。
7. bootstrap p-value。
8. 換手率。

---

## 7. 統計檢定

由於日頻股票預測訊號通常較弱，本研究進一步使用統計檢定確認績效是否可能只是隨機結果。

主要檢查：

1. D10-D1 平均報酬是否顯著大於 0。
2. 真實模型排序是否優於隨機排序。
3. 不同 seed 下結果是否穩定。
4. 不同測試期間下結果是否一致。
5. 扣除交易成本後績效是否仍存在。
